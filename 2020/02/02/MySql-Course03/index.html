    指定搜索条件。
    主要是对where子句学习，以及where子句与一些关键字和子句的综合使用。

    过滤数据

    where子句操作符

    操作符 说明
    = 等于
    <> 不等于
    != 不等于
    < 小于
    <= 小于等于
    > 大于
    >= 大于等于
    between 在指定的两个值之间

    where的简单应用

    select prod_name,prod_price
    from products
    where prod_price = 2.50;

    select prod_name,prod_price
    from products
    where prod_name = ‘fuses’;
    MySql在执行匹配时默认不区分大小写，所以fuses与FUSES匹配

    select prod_name,prod_price
    from products
    where prod_price < 10;

    select prod_name,prod_price
    from products
    where prod_price <= 10;

    select vend_id,prod_name
    from products
    where vend_id <> 1003;
    与
    select vend_id,prod_name
    from products
    where vend_id != 1003;
    相同

    select prod_name,prod_price
    from products
    where prod_price between 5.99 and 10;

    注：这里5.99和10都能被返回，所以between and 相当于 >= and <=

    空值检查

    NULL无值（no value），与字段0、空字符串或仅仅包含空格不同。

    select cust_id,cust_email
    from customers
    where cust_email is null;

    注：不能通过NULL值来过滤，从而返回具有NULL值的行。因为未知具有特殊含义，数据库不知道它们是否匹配，所以在匹配过滤或不匹配过滤时不返回它们。因此，在过滤数据时，一定要验证返回数据中确实给出了被过滤列具有NULL的行。
    注：输入的过滤参数类型需要与对应字段的数据类型一致，例如prod_price的数据类型为int型则输入参数为数字，不需要加引号。
    注：在同时使用order by和where时，order by应该位于where之后。

    数据过滤(过滤数据的进阶)

    主要是对AND、OR、IN、NOT关键字的组合使用。

    AND操作符

    select prod_id,prod_price,prod_name
    from products
    where vend_id = 1003 and prod_price <= 10;

    可以添加多个过滤条件，每添加一条就要使用一个AND。

    OR操作符

    select vend_id,prod_name,prod_price
    from products
    where vend_id = 1002 OR vend_id = 1003;

    计算次序

    列出价格为10美元（含）以上且由1002或1003制造的所有产品。

    select prod_name,prod_price
    from products
    where vend_id = 1002 or vend_id = 1003 and prod_price >= 10;

    如上所示，在结果中出现了价格低于10的产品。这是由于在检索时先做and计算，再做or计算。and的优先级要高于or。
    正确的语句：
    select prod_name,prod_price
    from products
    where (vend_id = 1002 or vend_id = 1003) and prod_price >= 10;

    加上括号后，计算次序发生改变，优先计算括号内。即先根据括号内的语句过滤数据，再根据括号外的语句进一步过滤。
    多条语句同时作用时，应该使用圆括号来明确计算的次序，以免产生歧义。

    IN操作符

    IN操作符用来指定条件范围，范围中的每个条件都可以进行匹配。IN取合法值的逗号分隔的清单，全都在括号中。

    select prod_name,prod_price
    from products
    where vend_id in (1002,1003)
    order by prod_name;

    由此可见IN操作符完成的功能与OR相同。

    IN操作符的优点：

    • 在使用长的合法选项清单时，IN操作符的语法更清楚直观
    • 计算次序更容易管理
    • 一般情况下IN操作符的执行速度比OR操作符更快
    • 可以包含其他SELECT语句

    NOT操作符

    用来否定它之后所跟的任何条件。

    select prod_name,prod_price
    from products
    where vend_id not in (1002,1003)
    order by prod_name;

    MySql支持使用NOT对IN、BETWEEN和EXISTS子句取反。

    用通配符进行过滤

    利用LIKE操作符进行

    百分号（%）通配符

    %表示任何字符出现任意次数。

    select prod_id,prod_name
    from products
    where prod_name like ‘jet%’;
    检索以产品名以jet开头的产品，搜索模式为‘jet%’

    通配符可以在搜索模式中任意位置使用
    select prod_id,prod_name
    from products
    where prod_name like ‘%anvil%’;

    注：尾空格可能会干扰通配符匹配。例如在保存anvil时，如果后面有一个或多个空格，则子句where prod_name like ‘%anvil’将不会匹配它们，因为l后面还有字符。
    注：通配符不能匹配用值NULL作为参数的行。

    下划线（_）通配符

    下划线通配符用法与百分号通配符一样，但只匹配单个字符

    select prod_id,prod_name
    from products
    where prod_name like ‘_ ton anvil’;

    使用通配符的技巧

    • 不要过度使用，因为通配符搜索的处理时间比之前学习的其他搜索所花的时间更长
    • 除非必要，否则不要把它们用在搜索模式的开始处
    • 仔细注意通配符的位置
