    文件上传漏洞概述
    因为业务功能需要，很多web站点都有文件上传的接口（比如注册时上传的头像等）
    由于没有对上传的文件类型进行严格的限制，导致攻击者可以上传一些恶意文件
    从而使得攻击者可以通过对该恶意文件的访问来控制整个web后台

    文件上传漏洞测试流程：

    • 对文件上传的地方按照要求上传文件，查看返回结果（路径，提示）
    • 尝试上传不同类型的“恶意文件”，查看结果
    • 查看html源码，看看是否是前端限制
    • 尝试使用不同方式进行绕过：黑名单绕过/MIME类型绕过/目录路径绕过等
    • 猜测或其他方式得到木马路径，连接，即可使用

    开撸

    知识补充：
    MIME(Multipurpose Internet Mail Extensions)多用途互联网邮件扩展类型。是设定某种扩展名的文件用一种应用程序来打开的方式类型，当该扩展名文件被访问的时候，浏览器会自动使用指定应用程序来打开。多用于指定一些客户端自定义的文件名，以及一些媒体文件打开方式。

    每个MIME类型由两部分组成，前面是数据的大类别，例如声音audio、图像image等，后面定义具体的种类。常见MIME类型：

    • 超文本标记语言文本 .html text/html
    • xml文档 .xml text/xml
    • XHTML文档 .xhtml application/xhtml+xml
    • 普通文本 .txt text/plain
    • RTF文本 .rtf application/rtf
    • PDF文档 .pdf application/pdf
    • Microsoft Word文件 .word application/msword
    • PNG图像 .png image/png
    • GIF图形 .gif image/gif
    • JPEG图形 .jpeg,.jpg image/jpeg
    • au声音文件 .au audio/basic
    • MIDI音乐文件 mid,.midi audio/midi,audio/x-midi
    • RealAudio音乐文件 .ra, .ram audio/x-pn-realaudio
    • MPEG文件 .mpg,.mpeg video/mpeg
    • AVI文件 .avi video/x-msvideo
    • GZIP文件 .gz application/x-gzip
    • TAR文件 .tar application/x-tar
    • 任意的二进制数据 application/octet-stream

    <?php @eval($_POST[cmd]);?>

    这里并没有对文件进行任何检查

    medium级别

    先上传一张正常的图片文件，能给成功上传返回路径
    上传个.php文件试试，发现上传失败，提示上传图片类型文件，观察网页源码分析并不是前端过滤
    接下来利用bp抓包分析

    将类型更改为这个即可成功上传

    <?php
if( isset( $_POST[ 'Upload' ] ) ) {
    // Where are we going to be writing to?
    $target_path  = DVWA_WEB_PAGE_TO_ROOT . "hackable/uploads/";
    $target_path .= basename( $_FILES[ 'uploaded' ][ 'name' ] );
    // File information
    $uploaded_name = $_FILES[ 'uploaded' ][ 'name' ];
    $uploaded_type = $_FILES[ 'uploaded' ][ 'type' ]; //MIME类型
    $uploaded_size = $_FILES[ 'uploaded' ][ 'size' ];
    // Is it an image?
    if( ( $uploaded_type == "image/jpeg" || $uploaded_type == "image/png" ) &&
        ( $uploaded_size < 100000 ) ) {

        // Can we move the file to the upload folder?
        if( !move_uploaded_file( $_FILES[ 'uploaded' ][ 'tmp_name' ], $target_path ) ) {
            // No
            echo '<pre>Your image was not uploaded.</pre>';
        }
        else {
            // Yes!
            echo "<pre>{$target_path} succesfully uploaded!</pre>";
        }
    }
    else {
        // Invalid file
        echo '<pre>Your image was not uploaded. We can only accept JPEG or PNG images.</pre>';
    }
}
?>

    if( ( $uploaded_type == “image/jpeg” || $uploaded_type == “image/png” ) && ( $uploaded_size < 100000 ) )
    关键代码，在后台检验类型，限制大小

    hight级别

    <?php
if( isset( $_POST[ 'Upload' ] ) ) {
    // Where are we going to be writing to?
    $target_path  = DVWA_WEB_PAGE_TO_ROOT . "hackable/uploads/";
    $target_path .= basename( $_FILES[ 'uploaded' ][ 'name' ] );
    // File information
    $uploaded_name = $_FILES[ 'uploaded' ][ 'name' ];
    $uploaded_ext  = substr( $uploaded_name, strrpos( $uploaded_name, '.' ) + 1);//获取上传文件的后缀名
    $uploaded_size = $_FILES[ 'uploaded' ][ 'size' ];
    $uploaded_tmp  = $_FILES[ 'uploaded' ][ 'tmp_name' ];
    // Is it an image?
    if( ( strtolower( $uploaded_ext ) == "jpg" || strtolower( $uploaded_ext ) == "jpeg" || strtolower( $uploaded_ext ) == "png" ) &&
        ( $uploaded_size < 100000 ) &&
        getimagesize( $uploaded_tmp ) ) {
        // Can we move the file to the upload folder?
        if( !move_uploaded_file( $uploaded_tmp, $target_path ) ) {
            // No
            echo '<pre>Your image was not uploaded.</pre>';
        }
        else {
            // Yes!
            echo "<pre>{$target_path} succesfully uploaded!</pre>";
        }
    }
    else {
        // Invalid file
        echo '<pre>Your image was not uploaded. We can only accept JPEG or PNG images.</pre>';
    }
}
?>

    这里进行了更严格的检查，getimagesize( $uploaded_tmp )这个函数返回结果中有文件大小和类型（即通过文件头来获取），绕过用这个函数来获取类型，从而判断是否是图片的化，仍然会存在问题，图片的文件头能被伪造。
    虽然可以突破上传，但是上传的文件没有很好的利用途径，需要结合其他漏洞进行结合利用（利用文件包含漏洞，可以参考文件包含漏洞部分的学习）

    文件上传漏洞绕过思路汇总

    客户端检查
    几乎等于摆设，通过抓包改包即可

    服务器端检查

    • 黑名单：尝试找到黑名单中的漏网之鱼，中间件的解析漏洞
    • 白名单：通常就需要使用到解析漏洞进行绕过
    • 文件完整性检查：文件头及相关信息检查，通过伪造文件头绕过

    防范措施

    不要在客户端实施上传限制策略

    通过服务器端对上传文件进行限制：

    • 进行多条件组合检查：比如文件的大小，路径，扩展名，文件类型，文件完整性
    • 对上传的文件在服务器上存储时进行重命名（制定合理的命名规则）
    • 对服务器端上传文件的目录进行权限控制（比如只读），限制执行权限带来的危害
